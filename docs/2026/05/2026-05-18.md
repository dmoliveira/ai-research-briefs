<h2 id="executive-summary">Executive Summary</h2>

The best work this week treats agents less like clever prompts and more like distributed software systems with state, scheduling, security boundaries, and measurable operating costs. FutureSim is the strongest benchmark idea because it evaluates adaptation against replayed real-world event streams rather than static question sets. The agent-search and web-agent papers are also important because they challenge two popular defaults: retrieval-heavy agent stacks and ReAct-style web browsing.

The second strong theme is memory quality. New work on modular knowledge updates, multimodal memory, recommender memory, and multi-granularity evidence retrieval all points in the same direction: useful AI products need memory systems that preserve provenance, handle partial observation, and expose enough structure to be audited. On the systems side, Amazon's architecture scaling-law article and the self-pruned KV-cache paper are useful reminders that model quality is increasingly constrained by serving economics, not only benchmark scores.

<h2 id="key-trends">Key Trends</h2>

<div class="brief-trend-grid">
  <div class="brief-trend-card">
    <p><strong>Agent evaluation is moving toward time and environment realism.</strong></p>
    <p>FutureSim, agentic search evaluation, and harness-safety auditing all focus on behavior over trajectories rather than final answers alone.</p>
  </div>
  <div class="brief-trend-card">
    <p><strong>Memory is becoming a product architecture problem.</strong></p>
    <p>The strongest memory papers separate stable preference, transient evidence, visual grounding, and knowledge updates instead of treating memory as a flat vector store.</p>
  </div>
  <div class="brief-trend-card">
    <p><strong>Web-agent security is converging on deterministic controls.</strong></p>
    <p>Plan-then-execute, harness-level auditing, identity, least privilege, and human review triggers all push enforcement out of the model and into the application layer.</p>
  </div>
  <div class="brief-trend-card">
    <p><strong>Efficiency work is becoming more architecture-aware.</strong></p>
    <p>Serving cost is being attacked through architecture search, async tool execution, and selective KV-cache writes rather than only smaller models.</p>
  </div>
</div>
