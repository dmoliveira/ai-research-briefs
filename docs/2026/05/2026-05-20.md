<h2 id="executive-summary">Executive Summary</h2>

The strongest theme this week is that agent capability is shifting from isolated model behavior toward the surrounding executable system. Code as Agent Harness gives the best conceptual map: code is becoming the substrate for reasoning, state, action, verification, and multi-agent coordination. EnvFactory, SkillGenBench, PopPy, and the three-layer safety-position paper make that same point from different angles: robust agents need environments, reusable procedures, runtime scheduling, and explicit safety contracts.

The second theme is practical scaling pressure. DashAttention and the readiness-driven pipeline runtime both target cost bottlenecks in long-context inference and large-model training. DBES examines whether mixture-of-experts routing is actually specializing in useful ways. These papers matter because model-quality improvements increasingly depend on whether systems can train, route, and serve workloads efficiently enough to use at product scale.

The most product-relevant item outside the agent cluster is Offline Contextual Bandits in the Presence of New Actions. Recommender and search systems constantly receive new items after logs were collected, and standard offline policy learning struggles with exactly that setting. TIGER-FG is also highly applied: it attacks visual e-commerce retrieval with a realistic 10M-pair benchmark and strong cluttered-layout gains.

  <h3>DashAttention: Differentiable and Adaptive Sparse Hierarchical Attention</h3>
  <p class="entry-summary">A long-context attention method that replaces fixed top-k block selection with adaptive sparse alpha-entmax routing and keeps sparse-to-dense attention differentiable.</p>
  <div class="entry-meta-grid">
    <div class="entry-meta-item"><span class="entry-meta-label">Published</span><span>2026-05-18</span></div>
    <div class="entry-meta-item"><span class="entry-meta-label">Authors</span><span>Yuxiang Huang, Nuno M. T. Goncalves, Federico Alvetreti, Lei Li, Xu Han, Edoardo M. Ponti, Andre F. T. Martins, Marcos V. Treviso</span></div>
    <div class="entry-meta-item"><span class="entry-meta-label">Affiliations</span><span>Not listed on the arXiv abstract page</span></div>
    <div class="entry-meta-item"><span class="entry-meta-label">Link</span><span><a class="entry-link" href="https://arxiv.org/abs/2605.18753">https://arxiv.org/abs/2605.18753</a></span></div>
    <div class="entry-meta-item"><span class="entry-meta-label">Tags</span><span>long context, sparse attention, inference efficiency, Triton</span></div>
    <div class="entry-meta-item"><span class="entry-meta-label">Reading profile</span><span>12-15 min &middot; Relevance 5/5 &middot; Quality 4/5</span></div>
  </div>
  <div class="entry-section-grid">
    <div class="entry-section"><p><span class="entry-section-title">Summary</span>DashAttention makes hierarchical sparse attention adaptive and differentiable by using alpha-entmax to select a query-dependent number of KV blocks.</p></div>
    <div class="entry-section"><p><span class="entry-section-title">Problem investigated</span>Fixed top-k sparse attention assumes every query needs the same number of blocks and blocks gradient flow between sparse and dense stages.</p></div>
    <div class="entry-section"><p><span class="entry-section-title">Context</span>Long-context LLMs need cheaper attention without sacrificing the rare but important tokens that determine answer quality.</p></div>
    <div class="entry-section"><p><span class="entry-section-title">Solutions explored</span>The method uses adaptive sparse routing as a prior for second-stage softmax attention and includes a GPU-aware Triton implementation.</p></div>
    <div class="entry-section"><p><span class="entry-section-title">Key results</span>The authors report comparable accuracy to full attention at 75% sparsity and a better Pareto frontier than NSA and InfLLMv2, especially at high sparsity.</p></div>
    <div class="entry-section"><p><span class="entry-section-title">Conclusion</span>Adaptive sparse attention can improve the cost-quality tradeoff for long-context modeling while preserving trainability.</p></div>
    <div class="entry-section"><p><span class="entry-section-title">My take</span>Technically strong and implementation-minded. The next proof point is whether it holds up in full pretraining or broad production inference workloads.</p></div>
