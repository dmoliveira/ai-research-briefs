<h2 id="executive-summary">Executive Summary</h2>

This week's strongest research signal is not another broad capability claim; it is a set of papers asking whether agents can be trusted when retrieval, tools, memory, and environments become messy. LiveBrowseComp is the best read-first item because it directly challenges a common assumption in search-agent evaluation: tool-using agents may be verifying what they already know rather than finding genuinely fresh evidence. VeriTrip extends the same concern into multimodal travel planning over unstructured web corpora.

Benchmark construction is also maturing. TASTE reverses the usual natural-language-to-tool-task pipeline and generates harder agent tasks from tool-sequence evolution. FeasiGen tests whether agents know when a task is impossible under the available tools. These papers matter because many production agents fail by continuing confidently through impossible or underspecified work.

Memory and personalization research is shifting from retrieval quality to debuggability and provenance. MemTrace introduces memory evolution graphs for tracing where memory systems corrupt or lose information, while Personal Visual Memory argues that personalized agents need durable visual facts, not just text memory and generic captions.

The safety cluster is unusually practical. The agent skill ecosystem report finds confirmed malicious payloads and widespread critical issues in public skill marketplaces. LACUNA proposes a typed programming model that rejects unsafe or invalid model-written actions before execution. Models That Know How Evaluations Are Designed Score Safer is a useful warning for anyone interpreting safety benchmark gains.

    <div class="entry-section"><p><span class="entry-section-title">Summary</span>The paper diagnoses intrinsic knowledge dependence in search agents and introduces LiveBrowseComp, a benchmark built from recent, less globally salient facts.</p></div>
    <div class="entry-section"><p><span class="entry-section-title">Problem investigated</span>Search-agent scores may reward confirming memorized facts rather than discovering answer-supporting evidence from the web.</p></div>

  <div class="entry-section-grid">
    <div class="entry-section"><p><span class="entry-section-title">Summary</span>TASTE constructs harder agent tasks by sampling and evolving valid tool sequences before producing the natural-language scenario.</p></div>
    <div class="entry-section"><p><span class="entry-section-title">Problem investigated</span>Manual benchmark authoring is expensive, and natural-language-first task design covers only a narrow slice of possible tool-use patterns.</p></div>
