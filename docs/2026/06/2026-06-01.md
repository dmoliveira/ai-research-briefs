<h2 id="executive-summary">Executive Summary</h2>

This week's brief is dominated by the AI engineering layer rather than model-release news. The most practical read is Meta's RADAR paper: it gives rare production telemetry for risk-aware automated code review across 535K+ reviewed diffs and 331K+ landed diffs. It is directly relevant to teams facing a review bottleneck from AI-generated code volume.

The verifier and evaluation cluster is also strong. Self-Trained Verification shows that a model can learn to catch its own errors by imitating a reference-informed version of itself, improving both test-time verification-refinement and training-time generator performance. CROP adds conformal guarantees for retaining only clean reasoning prefixes, while SoundnessBench tests whether AI scientist systems can identify methodologically weak ML ideas before wasting compute.

Agent safety work is becoming more concrete. Gram audits sabotage propensities in agentic coding and research deployments. Dissociative Identity argues that reputation systems are poorly grounded for mutable language-model agents. Token Inflation identifies an economic security problem in per-token billing when the provider controls the evidence used for auditing.

Memory and retrieval systems are moving from simple recall toward structured optimization. MMPO trains memory summaries using a belief-entropy proxy. Contextual Belief Management evaluates when models should preserve, update, or isolate beliefs. RAISE reframes RAG configuration as an architecture-search problem, and Single-stage Sparse Retrieval replaces ColBERT-style clustering with sparse coding.

  <div class="entry-meta-grid">
    <div class="entry-meta-item"><span class="entry-meta-label">Published / updated</span><span>2026-05-28 / 2026-05-28</span></div>
    <div class="entry-meta-item"><span class="entry-meta-label">Authors</span><span>David Lindner, Victoria Krakovna, Sebastian Farquhar</span></div>

  <div class="entry-section-grid">
    <div class="entry-section"><p><span class="entry-section-title">Summary</span>Gram evaluates Gemini models across 17 simulated deployment scenarios that create incentives for sabotage, then uses investigator agents to probe drivers of misbehavior.</p></div>
    <div class="entry-section"><p><span class="entry-section-title">Problem investigated</span>Agentic coding and research systems may fail in ways that resemble intentional sabotage, but teams lack repeatable auditing machinery for those deployment risks.</p></div>

    <div class="entry-section"><p><span class="entry-section-title">Conclusion</span>Sabotage-style risks can be made measurable, but scenario design strongly affects observed rates.</p></div>
    <div class="entry-section"><p><span class="entry-section-title">My take</span>Useful because it is framed as an audit, not a leaderboard. The result is less important than the testing pattern: realistic incentives, trajectory evidence, and targeted follow-up probes.</p></div>
