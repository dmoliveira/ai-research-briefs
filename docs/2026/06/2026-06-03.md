<h2 id="executive-summary">Executive Summary</h2>

This issue is about what breaks when agents leave clean demos. The best engineering read is <em>Monitoring Agentic Systems Before They're Reliable</em>: it argues that early production agents should be monitored for structural defects before teams expect task-level error detection to work. That framing is useful because many current deployments are still assemblies of brittle stages, tools, documents, policies, and model calls.

The agent security cluster is unusually concrete. <em>SkillHarm</em> turns third-party skills into a lifecycle attack surface, including persistent self-mutating poisoning. <em>Ghost Tool Calls</em> identifies a privacy leak created by speculative tool dispatch before the agent commits to a branch. <em>Tracking the Behavioral Trajectories of Adapting Agents</em> proposes a way to score whether edits to skill or memory files shift an agent toward sensitive-data-seeking behavior.

Evaluation work is becoming more operational. <em>AGENTCL</em> shows why naive task streams do not rigorously test continual learning in agents. <em>MCP-Persona</em> moves MCP evaluation toward personal and enterprise applications such as Slack and social tools. <em>K-BrowseComp</em> adds a Korean-context browsing benchmark where frontier models drop sharply, reinforcing the need for local, multilingual, fresh web-agent evaluation.

The systems papers are practical: <em>RASER</em> reduces multi-hop QA token cost by escalating only when cheap one-shot RAG looks unrecoverable, <em>SpectraMB</em> attacks noisy multi-behavior recommendation, <em>SubFit</em> compresses LLMs at attention/feedforward submodule granularity, and <em>SimSD</em> adapts speculative decoding to diffusion language models.

  <div class="entry-section-grid">
    <div class="entry-section"><p><span class="entry-section-title">Summary</span>The paper decomposes agent monitoring across quality, suitability, and efficiency, then across within-run, cross-run, and structural scopes.</p></div>
    <div class="entry-section"><p><span class="entry-section-title">Problem investigated</span>Early production agents often fail because of structural integration defects, making final-task error monitors noisy or misleading.</p></div>
    <div class="entry-section"><p><span class="entry-section-title">Context</span>Many deployed agent workflows are multi-stage document pipelines where deterministic defects, stochastic model behavior, and workflow gaps interact.</p></div>

    <div class="entry-section"><p><span class="entry-section-title">Conclusion</span>Monitor maturity should progress from structural characterization to task-error detection to reliability tracking as integration defects are resolved.</p></div>
    <div class="entry-section"><p><span class="entry-section-title">My take</span>This is the best operations read of the week. It matches how real agent deployments fail: first fix the workflow skeleton, then ask finer-grained accuracy questions.</p></div>

    <div class="entry-section"><p><span class="entry-section-title">Summary</span>The paper identifies ghost tool calls: speculative invocations issued to hide latency before the agent commits to the branch that would actually require them.</p></div>

    <div class="entry-section"><p><span class="entry-section-title">My take</span>Excellent systems-security paper. It exposes a subtle cost of making agents feel faster: privacy can be lost before an action "happens."</p></div>
