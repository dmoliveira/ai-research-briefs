<h2 id="executive-summary">Executive Summary</h2>

This week is about making agent behavior inspectable and trainable at the process layer. <em>Where Do Deep-Research Agents Go Wrong?</em> is the clearest read-first paper: it moves deep-research evaluation from final answers to span-level error localization over claims, evidence, and trajectory segments. That is closer to how product teams need to debug agents: identify the first harmful unsupported claim, not just whether the final report passed.

Search-agent training is becoming more concrete. <em>GrepSeek</em> trains agents to search raw corpora through constrained shell-style interaction instead of relying only on embedding retrievers. <em>Harness-1</em> shifts routine search bookkeeping into a stateful environment and lets the model decide what to search, keep, verify, or stop. <em>OpenWebRL</em> brings online multi-turn RL to visual web agents on live websites, which is important because static demonstrations age quickly.

The harness and skill papers reinforce a product-systems lesson: agent capability increasingly lives in editable external state. <em>COLLEAGUE.SKILL</em>, <em>Skill is Not One-Size-Fits-All</em>, <em>Harness Updating Is Not Harness Benefit</em>, and <em>Joint Agent Memory and Exploration Learning</em> all ask how skills, memories, and harness updates should be generated, adapted, trained, and evaluated.

Safety and efficiency remain tightly coupled to these systems. <em>Reproducing, Analyzing, and Detecting Reward Hacking in Rubric-Based Reinforcement Learning</em> provides a controlled setup for studying LLM-judge exploitation. <em>From Prompt Injection to Persistent Control</em> treats local workspace agents as persistent-control attack surfaces. On the systems side, <em>KVarN</em>, <em>Domino</em>, <em>Draft-OPD</em>, and <em>dMoE</em> target the memory and latency bottlenecks that determine whether these agents can run economically.

<h2 id="key-trends">Key Trends</h2>

<div class="brief-trend-grid">
  <div class="brief-trend-card">
    <p><strong>Agent evaluation is moving inside the trajectory.</strong></p>
    <p>Final-answer scoring is being supplemented by span localization, claim ledgers, verification records, and first-error analysis.</p>
  </div>
  <div class="brief-trend-card">
    <p><strong>Search agents are becoming trained systems, not prompt recipes.</strong></p>
    <p>GrepSeek, Harness-1, OpenWebRL, and LongTraceRL all use structured rollouts or RL to improve search behavior under real context and budget pressure.</p>
  </div>
  <div class="brief-trend-card">
    <p><strong>External harness state is now a core model interface.</strong></p>
    <p>Skills, memories, adapters, curated evidence pools, and editable figure plans are treated as persistent state that must be learned, governed, and secured.</p>
  </div>
  <div class="brief-trend-card">
    <p><strong>Inference efficiency is targeting reasoning workloads.</strong></p>
    <p>KV-cache quantization, speculative drafter training, dLLM routing, and compact RAG models are tuned for long decoding, not just short prefill benchmarks.</p>
  </div>
</div>
