<h2 id="executive-summary">Executive Summary</h2>

This week is about the operational layer around AI agents. <em>Evaluation Cards</em> is the best read for teams that publish, consume, or compare benchmark results: it turns scattered evaluation claims into structured records with provenance, reproducibility, documentation completeness, and score-comparability signals. <em>ComplexConstraints</em> is the most immediately useful training/evaluation paper because it argues that expert rubrics are not just reporting aids; they can become RL training signals for instruction following and enterprise agent tasks.

Personalized and interactive agents are also getting harder benchmarks. <em>iOSWorld</em> is a strong phone-agent benchmark because it tests persistent identity, personal data, multi-app tasks, and accessibility-tree variants rather than isolated UI chores. <em>OmniGameArena</em> extends VLM agent evaluation into real-time Unreal Engine games and tracks whether reflection-based skill prompts improve or generalize. These papers make the same point from different surfaces: single-shot benchmark scores are insufficient for agents that learn, personalize, and operate across stateful environments.

The infrastructure story is equally important. Hugging Face's OpenEnv post frames agentic RL as an interoperability problem between harnesses, environments, and trainers. <em>Claw-R1</em> makes a complementary data-management argument: multi-turn traces should be captured, inspected, curated, and batched as step-level assets. <em>CHAP</em> then extends the boundary to accountable human-agent workspaces where approvals, edits, handoffs, and rationales become replayable evidence.

The applied systems entries are practical. <em>Context Rot</em> names the stale-instructions problem in AI-assisted software development. <em>SIGA</em> shows that lightweight adapters can make coding agents operate scientific simulators. <em>FASE</em> proposes cheaper semantic-entropy estimation for generated code quality. OpenAI's <em>Dreaming</em> memory post is vendor-authored, but it gives concrete production concerns around freshness, correctness, and scalable personalization.

    <div class="entry-section"><p><span class="entry-section-title">Problem investigated</span>Scientific simulators have specialized input languages that are expensive for scientists and generic agents to learn from scratch.</p></div>

    <div class="entry-meta-item"><span class="entry-meta-label">Link</span><span><a class="entry-link" href="https://arxiv.org/abs/2606.09800">https://arxiv.org/abs/2606.09800</a></span></div>

  </div>
  <div class="entry-section-grid">
    <div class="entry-section"><p><span class="entry-section-title">Summary</span>FASE estimates uncertainty using minimum spanning trees over structural and semantic dissimilarity graphs rather than repeated LLM entailment checks.</p></div>
    <div class="entry-section"><p><span class="entry-section-title">Problem investigated</span>Semantic entropy is useful when ground-truth tests are absent, but LLM-driven equivalence checks can be too expensive for production code-generation workflows.</p></div>

    <div class="entry-section"><p><span class="entry-section-title">Conclusion</span>Smoother divergence regularization may be a better trust-region primitive than hard masks for LLM RL.</p></div>

  <h3>Steganography Without Modification: Hidden Communication via LLM Seeds</h3>
  <p class="entry-summary">A security paper showing that deterministic sampling seeds can act as hidden communication channels without changing model weights, code, or output distributions.</p>
  <div class="entry-meta-grid">
    <div class="entry-meta-item"><span class="entry-meta-label">Published / updated</span><span>2026-06-08 / 2026-06-08</span></div>
    <div class="entry-meta-item"><span class="entry-meta-label">Authors</span><span>Felix Mächtle, Jonas Sander, Sebastian Berndt, Ben Weimar, Nils Loose, Thomas Eisenbarth</span></div>
    <div class="entry-meta-item"><span class="entry-meta-label">Affiliations</span><span>Not listed in arXiv metadata; to appear at ARES 2026</span></div>
    <div class="entry-meta-item"><span class="entry-meta-label">Link</span><span><a class="entry-link" href="https://arxiv.org/abs/2606.09135">https://arxiv.org/abs/2606.09135</a></span></div>
    <div class="entry-meta-item"><span class="entry-meta-label">Tags</span><span>LLM security, steganography, sampling, seeds, covert channels</span></div>
    <div class="entry-meta-item"><span class="entry-meta-label">Reading profile</span><span>14-20 min &middot; Relevance 4/5 &middot; Quality 4/5</span></div>
  </div>
  <div class="entry-section-grid">
    <div class="entry-section"><p><span class="entry-section-title">Summary</span>The paper shows that a sender can encode a secret message in the PRNG seed used for LLM sampling, while a receiver reconstructs token-level probability intervals from generated text to recover the seed.</p></div>
    <div class="entry-section"><p><span class="entry-section-title">Problem investigated</span>LLM inference stacks may carry covert channels even when outputs are distribution-preserving and no model or sampler code is modified.</p></div>
    <div class="entry-section"><p><span class="entry-section-title">Context</span>Organizations increasingly rely on model outputs crossing trust boundaries, where covert communication and data exfiltration risks matter.</p></div>
    <div class="entry-section"><p><span class="entry-section-title">Solutions explored</span>The authors formalize known-prompt and unknown-prompt recovery settings and test seed recovery across six model families and five text domains.</p></div>
    <div class="entry-section"><p><span class="entry-section-title">Key results</span>In the known-prompt setting, the abstract reports up to 100% recovery of 32-bit seeds within 300 tokens and under 35 seconds on a single GPU; unknown-prompt recovery approaches near-perfect accuracy at 600-800 tokens.</p></div>
    <div class="entry-section"><p><span class="entry-section-title">Conclusion</span>Seed choice can be a hidden channel, and ignorance of the prompt is not a sufficient security assumption.</p></div>
    <div class="entry-section"><p><span class="entry-section-title">My take</span>This is a concrete, surprising threat model for controlled-generation systems. It deserves attention from safety, platform, and audit-log teams.</p></div>
