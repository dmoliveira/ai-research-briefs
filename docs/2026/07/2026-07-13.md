<h2 id="executive-summary">Executive Summary</h2>

The strongest work this week is about making AI systems inspectable, controllable, and operationally useful. Anthropic's <em>A global workspace in language models</em> is the most important interpretability item: it argues that Claude has a small set of internal representations that behave like a reportable, controllable reasoning workspace. Anthropic and AE Studio's <em>An off switch for dual-use knowledge</em> is equally practical for safety engineering because it explores modular routing of sensitive knowledge during training, so capabilities can later be removed without retraining separate models.

Agent work is moving from benchmark scoring into diagnosis and operations. <em>AgentLocate</em>, <em>STRACE</em>, <em>Think Big, Search Small</em>, <em>Agora</em>, and <em>Harness Engineering for Auditable Enterprise LLM Agents</em> all study the same production question from different angles: once an agent system fails, how do we identify which component failed, extract reusable improvement signals, route work to the right capability, and keep behavior auditable across model substitutions?

AI-native data workflows are becoming a clear product-systems theme. <em>Spider 2.0-AIFunc</em> updates text-to-SQL evaluation for data warehouses where LLM functions are part of SQL itself. Microsoft's <em>Flint</em> turns visualization into a compact, semantic intermediate language that agents can generate and validate. Google's <em>SensorFM</em> shows the scale of foundation models for sensor data and is notable for its agentic code-search classroom that builds prediction heads.

Recommendation and retrieval work is also shifting from static rankers toward semantic planning, multimodal memory, and generative retrieval. <em>From Raw IDs to Semantic Planning</em> is conceptually useful for recommender-system architects, while <em>MMEACR</em> and <em>DaV-Gen</em> explore concrete designs for agentic, multimodal, and end-to-end search/recommendation stacks.
